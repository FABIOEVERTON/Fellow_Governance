⚖️ PCG: Governance Control Plane (3 estágios)

Admissibility → Binding Validation → Execution 
1️⃣ ADMISSIBILITY: A proposta é legal? (PL 2338, LGPD, etc)
   • freshness < 24h → PASS
   • completeness: all_required_fields → PASS
   • provenance via hash chain → PASS
   • confidence ≥ 0.85 → PASS

2️⃣ BINDING VALIDATION: Autoridade existe no momento da execução?
   • opa eval --input context.json policy.rego (timeout 2s)
   • Retorna {allow: bool, trace: [], token: jwt}

3️⃣ EXECUTION: Só após 1 e 2, ação se torna real
   • Se violado → PCG halt + audit trail

🔗 AEC: Controlled Execution Architecture + SHA-256 Hash Chain

Cadeia imutável + Procedimento de recuperação 
prev_hash = SHA256(previous.payload_hash + previous.timestamp)

RECUPERAÇÃO DE QUEBRA:
1. LOCK_SESSION → bloqueia escritas
2. P0 alert → recovery_token (offline JWT, 1h)
3. new_prev_hash = SHA256("RECOVERY_AFTER_BREAK_" + timestamp + token)
4. Registro em recovery_events (imutável)
5. Validação da nova cadeia → release writes

OPA Fallback - 4 modos de operação 
Normal: OPA responde em <2s → validação completa
Degraded: OPA timeout 2s-5s → último cache + degraded_mode=true
